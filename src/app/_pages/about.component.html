brief, build & manage emails

Coordinate layouts, assets & approvals process for modular emails

Campaign Supernova provides a simple user interface to devise and brief layouts for email campaign templates; store and edit all your assets; and get layouts and assets reviewed and approved by other members of your team.

Data for each email gets stored to Firebase's database and can be exported ready to compile an html email to your exact spec.

So this isn't an email build tool?

CS functions as a companion tool to an email builder, specifically one that generates and populates email templates based on a set of reference data.

Each time you compose a layout and collate assets for an email you are not just coordinating your resources but compiling the data which can be directly referenced to build that email.

I want an email build tool!

Campaign Builder is a ‘one click’ email builder that can do the rest.*

The modules available in Campaign Supernova's UI correspond to html modules available in Campaign Builder's library.

If you want to build a template based on the campaign data you have set up in CS - then Campaign Builder can make that happen.

*Campaign Builder is in beta and currently runs as a templating tool via local server. It can be used by any developer familar with npm and download, installed and run using the command line.

Whether briefing your own or another organisation CS can help maximise workflow and minimise errors that occur when working with static briefing documents; exchanging wireframes and assets via email or server; collating feedback and changes over email chain or hosted documents.

Controls...

Campaign Supernova allows registered users to be assigned one of three different types of profile when initiating a campaign. Each of these profiles have a unique set of privileges when accessing a campaign:

Author

  • • Create, edit or delete campaign modules
  • • Request QA
  • • Request final approval from the owner

The Author's access to editing controls is restricted once the campaign has been approved by its Owner.

QA

  • • Approve or reject campagin modules
  • • Apply a red flag to any data field FAO the author
  • • Register feedback for any module

Only when the QA has approved all campaign modules and data can the Author request approval from the Owner.

Owner

  • • Approve or reject a campaign
  • • Register feedback regarding the campaign
  • • Apply a red flag to any data field or module FAO the author

Only when the Owner has approved an email campaign is its data made accesible to generate the final template.
